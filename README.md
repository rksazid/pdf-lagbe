# pdf-lagbe

HTML-to-PDF API service with full JavaScript execution and 3-layer security. Send any HTML ‚Äî including Chart.js, QRCode.js, Tailwind CSS, custom fonts, and images ‚Äî and get back a rendered PDF.

Built with Node.js, TypeScript, Express, and Puppeteer. Deploys on Render free tier (512 MB RAM).

## Features

- **Full HTML/CSS/JS rendering** ‚Äî Charts, QR codes, dynamic tables, canvas, SVG all work
- **CDN support** ‚Äî Load scripts/styles from jsdelivr, cdnjs, unpkg, Google Fonts, Tailwind CSS
- **Images from any HTTPS host** ‚Äî Clients can embed images from their own servers
- **3-layer security** ‚Äî DOMPurify sanitization, runtime API overrides, network interception
- **Configurable output** ‚Äî Page format, orientation, margins, scale, headers/footers
- **Rate limiting** ‚Äî Per-IP request throttling
- **Health checks** ‚Äî Browser status, memory usage, uptime
- **Structured logging** ‚Äî JSON logs in production, pretty console in development

## Quick Start

**Prerequisites:** Node.js >= 20

```bash
git clone https://github.com/rksazid/pdf-lagbe.git
cd pdf-lagbe
npm install
npm run dev
```

Generate a PDF:

```bash
curl -X POST http://localhost:3000/api/v1/pdf \
  -H "Content-Type: application/json" \
  -d '{"html": "<h1>Hello World</h1><p>Generated by pdf-lagbe</p>"}' \
  --output hello.pdf
```

## API Reference

### `POST /api/v1/pdf`

Generate a PDF from HTML content.

**Request body** (JSON):

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `html` | string | *required* | HTML content (max 2 MB) |
| `format` | string | `"A4"` | `A4`, `Letter`, `A3`, `Legal`, `Tabloid` |
| `landscape` | boolean | `false` | Landscape orientation |
| `margin` | object | `1cm` all sides | `{ top, right, bottom, left }` as CSS strings |
| `printBackground` | boolean | `true` | Include background colors/images |
| `scale` | number | `1` | Scale factor (0.1 ‚Äì 2.0) |
| `displayHeaderFooter` | boolean | `false` | Show header and footer |
| `headerTemplate` | string | ‚Äî | Header HTML (max 10 KB) |
| `footerTemplate` | string | ‚Äî | Footer HTML (max 10 KB) |
| `preferCSSPageSize` | boolean | `false` | Use `@page` CSS size over `format` |
| `waitForSelector` | string | ‚Äî | CSS selector to wait for before rendering |
| `waitForTimeout` | number | ‚Äî | Extra wait in ms (0 ‚Äì 5000) for JS to finish |

**Response:**

- `200` ‚Äî PDF binary (`Content-Type: application/pdf`, `X-Generation-Time` header)
- `400` ‚Äî Validation error (missing/invalid fields)
- `408` ‚Äî Render timeout
- `413` ‚Äî HTML exceeds 2 MB
- `429` ‚Äî Rate limited
- `503` ‚Äî At capacity (includes `Retry-After` header)

## Examples

### Simple styled HTML

```bash
curl -X POST http://localhost:3000/api/v1/pdf \
  -H "Content-Type: application/json" \
  -d '{
    "html": "<html><head><style>body{font-family:sans-serif;padding:2rem}h1{color:#2563eb}</style></head><body><h1>Invoice #1234</h1><p>Amount: $500.00</p></body></html>",
    "format": "Letter"
  }' --output invoice.pdf
```

### Chart.js + QR code from CDN

```bash
curl -X POST http://localhost:3000/api/v1/pdf \
  -H "Content-Type: application/json" \
  -d '{
    "html": "<html><head><script src=\"https://cdn.jsdelivr.net/npm/chart.js\"></script><script src=\"https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js\"></script></head><body><canvas id=\"c\"></canvas><div id=\"qr\"></div><script>new Chart(document.getElementById(\"c\"),{type:\"bar\",data:{labels:[\"A\",\"B\",\"C\"],datasets:[{data:[10,20,15]}]},options:{animation:false}});new QRCode(document.getElementById(\"qr\"),{text:\"https://example.com\",width:128,height:128});</script></body></html>",
    "waitForTimeout": 2000
  }' --output chart.pdf
```

### Google Fonts + external image

```bash
curl -X POST http://localhost:3000/api/v1/pdf \
  -H "Content-Type: application/json" \
  -d '{
    "html": "<html><head><link href=\"https://fonts.googleapis.com/css2?family=Poppins:wght@400;700&display=swap\" rel=\"stylesheet\"><style>body{font-family:Poppins,sans-serif;padding:2rem}</style></head><body><h1>Report</h1><img src=\"https://res.cloudinary.com/demo/image/upload/sample.jpg\" width=\"300\"></body></html>"
  }' --output report.pdf
```

## Configuration

All settings are configured via environment variables.

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `3000` | Server port |
| `NODE_ENV` | `development` | `production` enables JSON logging |
| `PDF_TIMEOUT` | `10000` | PDF generation timeout (ms) |
| `MAX_HTML_SIZE` | `2097152` | Max HTML body size in bytes (2 MB) |
| `DEFAULT_FORMAT` | `A4` | Default page format |
| `MAX_CONCURRENT_PAGES` | `2` | Max simultaneous browser pages |
| `RATE_LIMIT_WINDOW` | `60000` | Rate limit window (ms) |
| `RATE_LIMIT_MAX` | `10` | Max requests per window per IP |
| `JS_EXECUTION_TIMEOUT` | `5000` | Timeout for `waitForSelector` (ms) |
| `ALLOWED_CDN_DOMAINS` | ‚Äî | Extra CDN domains (comma-separated) |

**Default trusted CDN domains** (always included):

- `cdn.jsdelivr.net`
- `cdnjs.cloudflare.com`
- `unpkg.com`
- `fonts.googleapis.com`
- `fonts.gstatic.com`
- `cdn.tailwindcss.com`

Add more via `ALLOWED_CDN_DOMAINS=kit.fontawesome.com,stackpath.bootstrapcdn.com`.

## Security Architecture

Every PDF request passes through 3 independent security layers:

```
HTML Input
  ‚îÇ
  ‚îú‚îÄ Layer 1: DOMPurify Sanitization
  ‚îÇ   ‚îú‚îÄ‚îÄ Strips: event handlers (onclick, onerror, ...), javascript: URLs
  ‚îÇ   ‚îú‚îÄ‚îÄ Strips: <iframe>, <object>, <embed>, <applet>, <form>
  ‚îÇ   ‚îî‚îÄ‚îÄ Preserves: <script>, <style>, <link>, SVG, canvas, data-* attrs
  ‚îÇ
  ‚îú‚îÄ Layer 2: Runtime API Overrides (evaluateOnNewDocument)
  ‚îÇ   ‚îú‚îÄ‚îÄ Blocks: fetch, XMLHttpRequest, WebSocket, EventSource, sendBeacon
  ‚îÇ   ‚îú‚îÄ‚îÄ Blocks: eval, document.cookie, localStorage, sessionStorage
  ‚îÇ   ‚îú‚îÄ‚îÄ Blocks: window.open
  ‚îÇ   ‚îî‚îÄ‚îÄ Preserves: DOM APIs, Canvas, setTimeout, Promise, Math, Date
  ‚îÇ
  ‚îî‚îÄ Layer 3: Network Interception (Chromium DevTools Protocol)
      ‚îú‚îÄ‚îÄ Allows: data: and blob: URIs
      ‚îú‚îÄ‚îÄ Allows: images from any HTTPS host
      ‚îú‚îÄ‚îÄ Allows: scripts/styles/fonts from trusted CDN domains only
      ‚îú‚îÄ‚îÄ Blocks: all other network requests (XHR, fetch, WebSocket, etc.)
      ‚îî‚îÄ‚îÄ URL length cap: 2048 chars (anti-exfiltration)
```

Even if a malicious script bypasses Layer 1, Layer 2 disables dangerous APIs and Layer 3 blocks all unauthorized network traffic at the browser level.

## Deployment (Render)

The project includes `render.yaml` for one-click deployment on Render.

```yaml
services:
  - type: web
    name: pdf-lagbe
    runtime: node
    plan: free
    buildCommand: chmod +x render-build.sh && ./render-build.sh
    startCommand: node dist/index.js
    healthCheckPath: /health
```

**Important:** `PUPPETEER_CACHE_DIR` must point inside the project directory (`/opt/render/project/src/.puppeteer`). The default Puppeteer cache path (`/opt/render/.cache/`) is not available at runtime on Render.

**Memory budget (512 MB free tier):**

```
512 MB total
‚îú‚îÄ‚îÄ OS + Node.js         ~60 MB
‚îú‚îÄ‚îÄ Chromium             ~150 MB
‚îú‚îÄ‚îÄ 2 concurrent pages   ~100 MB
‚îú‚îÄ‚îÄ Safety margin        ~50 MB
‚îî‚îÄ‚îÄ Remaining            ~150 MB headroom
```
